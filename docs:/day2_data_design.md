# Day2 UIとデータの関係

## 事実

- cardの構造を作成した
- cardはtitle / image / date / descriptionのデータを表示する
- card構造をHTMLで作成した

## 理解

- articleは1つの記事データ
- articlesは複数の記事データの集合
- cardはUIでありarticleのデータを表示している
- UIはデータをユーザーに見せる仕組み

## 本質

UI = データをユーザーに見せる仕組み

## 判断

- UIは見た目ではなく、データから考える

## エラー

- UIとデータの違いが曖昧だった
- HTMLタグ（h2やp）をデータとして捉えていた

## 原因

- 「表示方法（UI）」と「中身（データ）」を区別できていなかった

## 再発防止

- データ（article）とUI（card）を必ず分けて考える

## 再現性

- UIを見るときは「どのデータを表示しているか」を必ず考える

- ループ（for文）に進む
