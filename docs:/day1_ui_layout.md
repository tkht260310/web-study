# DAY1 UI設計とレイアウト

##　事実

- cardの構造を作成した
- cardを３つ並べて表示した
- flexを使って横並びにした
- paddingやwidthを調整した

## 理解

- cardはUIであり、１つのデータを表示する単位
- flexを使うと要素を横並びにできる
- flexは可変、widthは固定
- 子要素は親要素の影響を受ける

## 本質

- UIは構造（中のデータの並びと関係）とレイアウト（どう並べるか）で成立

## 判断

- 見た目（レイアウト）を整える前に構造（データの並びと関係）を作るべき

## エラー

- カードを横並びにしたとき、カードの幅が揃わず崩れた
- タイトルの長いカードだけ横幅が広がった
- 画像サイズがカードごとバラバラになった

## 原因

- .card に flex: 1 1 250px を指定していたため、カードの幅が可変になっていた
- タイトルの文字数が多いカードだけ横幅が広がっていた
- img に width: 100% を指定していなかったため、画像サイズがカードに揃っていなかった

## 再発防止

- カード幅を固定したい場合は width を使う
- 可変にする場合は flex の挙動を前提に設計する
- 画像は必ず width: 100% を指定する

## 再現性

- UIを作るときは「構造→レイアウト→調整」の順で考える

## 次の行動

- UIとデータの関係に進む
