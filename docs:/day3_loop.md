# Day3 ループ（for文）の理解

## 事実

- cardを3つ表示した
- flexで横並びにした
- 同じcard構造が複数並んでいることを確認した

## 理解

- articleは1つのデータ
- articlesは複数のデータの集合
- cardはarticlesの中の1つのarticleデータを使って作られる
- データの数だけUI（card）が繰り返し表示される
- for文は、データの数だけ同じ処理を繰り返す仕組み

## 本質

UI = データ × 繰り返し

## 判断

- cardを何度も手で書くのではなく、データを1つずつ取り出して繰り返し表示すると考える

## エラー

- articlesとarticleの違いが曖昧だった
- cardがデータなのかUIなのか一時的に混乱した

## 原因

- 複数データの集合と、1つのデータの区別が曖昧だった
- データとUIの役割を分けて考えきれていなかった

## 再発防止

- articlesは複数データ、articleはその中の1件と分けて考える
- cardはUI、articleはデータと明確に区別する

## 再現性

- 繰り返しのあるUIを見たら、まず「どのデータを何回表示しているか」を考える

## 次の行動

- 条件分岐（if）に進む
