## 学んだこと詳細

### SSHキーと個人アクセストークンの違い

- 個人アクセストークン = 合言葉。有効期限があるので切れたら再発行が必要
- SSHキー = 特別な鍵と鍵穴。一回設定したら期限なしでずっと使える
- エンジニアとしてSSHキーが主流

### 秘密鍵と公開鍵の仕組み

- ssh-keygenで2つの鍵が作られる
  - `id_ed25519`（秘密鍵）← 絶対に誰にも見せたらいかん
  - `id_ed25519.pub`（公開鍵）← GitHubに登録する用
- 場所：`~/.ssh/` の中に保存されとる
- 中身を見るコマンド：`cat ~/.ssh/id_ed25519.pub`

### ssh-agentの役割

- ssh-agent = 鍵の一時預かり所（管理人）
- これがないとgit pushのたびにパスフレーズを毎回入力せんといかん
- 起動コマンド：`eval "$(ssh-agent -s)"`
- 鍵を預けるコマンド：`ssh-add ~/.ssh/id_ed25519`

### GitHubへのSSH公開鍵登録手順

1. `cat ~/.ssh/id_ed25519.pub` で公開鍵をコピー
2. GitHub → Settings → SSH and GPG keys
3. 「New SSH key」で貼り付けて登録
4. `ssh -T git@github.com` で接続テスト
5. `Hi ユーザー名! You've successfully authenticated...` が出たら成功

### リポジトリをHTTPS→SSH接続に切り替え

- 確認コマンド：`git remote -v`
- 切り替えコマンド：`git remote set-url origin git@github.com:ユーザー名/リポジトリ名.git`
- GitHubのCodeボタン→SSHタブでURLを確認できる

### git push / git pullの方向

パソコン → GitHub git push（送る）
パソコン ← GitHub git pull（持ってくる）

### master と main の違い

- 2020年にGitHubがmasterからmainに変更
- 理由：差別的な言葉を連想させるため
- Progateではmasterが出てくるが、実際はmainが主流
- どちらも同じもの、名前が違うだけ

### origin masterのフルパス記述

- `git push origin master` の意味
  - origin = GitHub（リモートリポジトリの名前）
  - master = ブランチ名（作業ライン）
- 慣れるまではフルで書く習慣をつける

### PublicとPrivateリポジトリの違い

- Public = 誰でも見える。採用担当にコードを見てもらえる
- Private = 自分だけ。コミット履歴がデフォルトで草に反映されない
- 学習用リポジトリはPublicのままでOK
- ただしコードにパスワード・APIキー・個人情報は絶対に書かない

### SSH configファイルの設定

- configファイル = SSH接続の設定を書いておくファイル
- 場所：`~/.ssh/config`
- 中身を見るコマンド：`cat ~/.ssh/config`
- 編集コマンド：`nano ~/.ssh/config`

#### ssh-agent方式とconfigファイル方式の違い

| 方式               | 特徴                                          |
| ------------------ | --------------------------------------------- |
| ssh-agent方式      | 起動するたびにssh-addで鍵を登録する必要がある |
| configファイル方式 | 設定を書いておくだけで毎回自動で読み込まれる  |

#### configファイルの設定内容

Host github.com ← GitHubだけに適用（\*にすると全サーバーに適用）
AddKeysToAgent yes ← 鍵を自動でssh-agentに登録する
UseKeychain yes ← Macのキーチェーンに鍵を保存する（再起動後もパスフレーズ不要）
IdentityFile ~/.ssh/id_ed25519 ← 使う鍵はこれ

#### `*`（ワイルドカード）について

- `Host *` = すべてのサーバーに適用
- `Host github.com` = GitHubだけに適用
- 個人PCでGitHubしか使わんなら `github.com` に限定する方が明示的でわかりやすい

#### nanoエディタの操作方法

- `nano ~/.ssh/config` でファイルを開く
- 編集が終わったら `Ctrl + X` → `Y` → `Enter` で保存
