#### Git・ターミナル

| コマンド | 意味 | 使うとき |
| "git add ファイル名" | コミット対象に選ぶ | ファイル変更時 |
| "git add ." | 全ファイルを対象に | まとめてaddしたいとき |
| "git commit -m """ | ローカルに記録 | addの後 |
| "git push" | GitHubに送信 | commitのあと |
| "git status" | 現在の状態の確認 | 変更状況を把握したいとき |
| "git rm --cached" | Git管理から外す | 不要ファイルを除外するとき |
| "git commit --amend" | 直前のcommit修正 | commit文を間違えたとき |
| "git push --force" | 強制push | amend五にpushするとき |
| "touch ファイル名" | ファイル作成 | 新しいファイル作成が必要なとき |
| "cd フォルダ名"| フォルダ移動 | 作業場所を変えるとき |
| "cd .." | １つ上に戻る | 親フォルダに戻るとき |
| "ls" | 中身を表示 | ファイルの存在確認 |　
| "pwd" | 現在地を確認 | 今どこにいるか確認 |
| "cat ファイル名" | 中身を表示 | ファイル内容確認 |
| "node ファイル名.js" | JSファイル実行 | 動作確認 |

## 詰まったこと・気づき

- docs/はフォルダの区切りを表す(docs/ファイル名　= docsフォルダの中のファイル)
- git rm --cachedはgit管理からファイルを除外するコマンド（DS_Store対策）
- git commit --amendはpush前のcommit文修正に使う
- push後にファイル修正するなら普通のgit pushでOK、force不要
- マークダウンの表記法（||で区切るとテーブル＝表になる）
